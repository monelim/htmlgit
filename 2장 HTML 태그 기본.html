    • 요소 : 태그를 사용해 만들어진 객체

      • 예) HTML5 요소 - 시작 태그와 끝 태그를 별도로 입혁하는 요소

      (h1 title="headeer") 속성블록 Hello HTML5(/h1) 내부문자
      title : 속성 이름 header : 속성 값

HTML5 페이지 구조

    lang 속성

    • 검색 엔진이 웹 페이지를 탐색할 때 해당 웹 페이지가 어떤 언어로 만들어져 있는지 쉽게 인식하게 만든다.
    • html 태그에 입력 ex) (html lang="ko")
    • 전 세계적인 데이터 네트워크 구축을 위해서는 lang 속성을 입력하는 것이 좋다.

    head 태그 내부에 넣을 수 있는 태그

    • meta : 웹 페이지에 추가 정보를 전달
    • script : 웹 페이지에 스크립트 추가
    • link : 웹 페이지에 다른 파일 추가
    • style : 웹 페이지에 스타일시트 추가
    • base : 웹 페이지의 기본 경로 지정
    • 검색 엔진이 웹 페이지를 탐색할 때 해당 웹 페이지
    • title : 웹 페이지의 제목 (웹 브라우저의 상단에 표시된다.)

      • - title 태그도 html 태그의 lang 속성처럼 검색 엔진에게 웹 페이지의 제목과 관련된 정보를 주는 데 사용되므로 입력하는 것이 좋다.

글자 형태

    제목

    HTML5의 대표적인 글자 태그는 제목을 입력할 때 사용하는 제목 글자 태그

      - 제목 글자 태그 -

      일반적으로 웹 페이지를 만들 때는 h1 태그부터 h3 태그까지 사용

    • h1 : 첫 번째로 큰 제목 글자 태그
    • h2 : 두 번째로 큰 제목 글자 태그
    • h3 : 세 번째로 큰 제목 글자 태그
    • h4 : 네 번째로 큰 제목 글자 태그
    • h5 : 다섯 번째로 큰 제목 글자 태그
    • h6 : 여섯 번째로 큰 제목 글자 태그

    본문

      - 본문 태그(1)

    • p : 본문 글자 태그, 하나의 단락을 만들 수 있다.

      • - 본문 태그(2)

    • br : 줄바꿈 태그 (개행)
    • hr : 수평 줄 태그 (가로로 그어지는 줄)

    • >>>>>>>>>>>> 조직화된 문서 형태를 가질 수 있는 이유 : 앵커 태그 때문
    • - a : 앵커 태그 (a 태그의 href 속성을 사용하여 이동하고자 하는 웹 페이지를 지정)
      • 하이퍼링크 태그 (body)
      (a href="htpp://hanbit.co.kr")Hanbit(/a)(br/)

    • 하이퍼링크 기능을 제거해도 웹 표준을 따르려면 a 태그에 href 속성을 반드시 입력해야 한다.
    • 웹 표준을 지키면서 이동하지 않는 a 태그를 만들 때 : href 속성에 #을 입력 >> 이를 빈 링크라고 부른다.
      • (li)(a href="#")언론사 전체보기(/a)

오디오

    : 웹 브라우저에서 플러그인의 도움 없이 음악을 재생할 수 있게 만들어주는 HTML5 태그

    ( 이 기능은 HTML5에서 추가된 기능이므로 인터넷 익스플로러 8이하에서는 사용할 수 없음 )

    audio 태그

      - audio 태그의 속성 -

    • scr : 음악 파일 경로 지정
    • preload : 음악 재생 전 모두 불러올지 지정
    • autoplay : 음악을 자동 재생할지 지정
    • loop : 음악을 반복할지 지정
    • controls : 음악 재생 도구를 출력할지 지정


      • 다음 2가지 방법 많이 사용

      - audio 태그의 속성 표기 방법(1) - XHTML5

      - audio 태그의 속성 표기 방법(2) - HTML5

    source 태그

      audio 태그 또는 video 태그 안에 입력하는 태그

      웹 브라우저마다 지원하는 음악 파일 형식이 미세하게 달라 실행되지 않을 수도 있기 때문에 사용

      ※ source 태그의 type 속성은 입력하지 않아도 상관없지만 음악 파일을 내려받은 뒤에 재생 가능한 파일인지 확인하므로 트래픽이 낭비된다.

      따라서 type 속성을 꼭 지정해주자 !

비디오 태그

    video 태그

      - video 태그의 속성 -

    • src : 비디오 파일의 경로 지정
    • poster : 비디오 준비 중일 때의 파일 경로 지정
    • preload : 비디오를 재생하기 전에 모두 불러올지 지정
    • autoplay : 비디오를 자동 재생할지 지정
    • loop : 비디오를 반복할지 지정
    • controls : 비디오 재생 도구를 출력할지 지정
    • width : 비디오의 너비 지정
    • height : 비디오의 높이 지정

      • video 태그도 audio 태그처럼 웹 브라우저마다 지원하는 비디오 파일 형식이 달라 실행되지 않을 수도 있기 때문에 source 태그를 사용



      ※poster 속성에 width와 height를 지정하지 않아도 이미지 생성시 설정해 주었기때문에 괜찮지만, 이미지 지정시 설정하지 않으면 크기가 얼마인지 이미지 위에 숫자가 나타나지 않는다.


      ※웹 브라우저마다 표시되는 video 태그의 형태가 일관되지 않으므로 문제가 생기거나 익스플로러 8 이하에서는 video 태그가 동작하지 않는다.

      이러한 문제를 해결하는 간단한 플로그인 : video.js 클로그인

      만들어진 스타일시트와 자바스크립트를 활용 -

      head 태그에 link 태그와 script 태그를 입력 >> video.js 플로그인의 홈페이지에서 복사

      이제 모든 웹에서 실행가능하며 익스플로러8에서는 자동으로 플래시에 담겨 동영상이 재생된다.

    입력 양식 개요

      : 사용자에게 입력받는 공간 ex) 회원 가입 양식

      • action : 입력 데이터의 전달 위치 지정
      • method : 입력 데이터의 전달 방식 선택

        • mehtod 속성 중 자주 쓰이는 것

      • GET : 주소에 데이터를 입력해서 보내는 방식(url변경), 한정적 크기
      • POST : 비밀스럽게 데이터를 전달하는 방식(url변경 x), 용량 제한 없음

    HTML5 시멘틱 구조 태그

      시멘틱 웹 : 어떠한 태그가 어떠한 기능을 하는지 분별할 수 있고, 웹 페이지에서 데이터를 효율적으로 추출할 수 있도록 특정한 데이터에 의미를 부여해서 웹 페이지를 만드는 것

      - HTML5 시멘틱 구조 태그 -

      • header : 헤더를 의미
      • nav : 내비게이션을 의미
      • aside : 사이드에 위치하는 공간을 의미
      • section : 여러 중심 내용을 감싸는 공간을 의미
      • article : 글자가 많이 들어가는 부분을 의미
      • footer : 푸터를 의미

      ※ 시멘틱 태그는 모두 div 태그와 같은 기능을 수행하는 태그

      ※ 태그 : 검색 엔진이나 그 이외의 기계적인 동작들이 웹 페이지를 쉽게 이해할 수 있게 하는 데 의미가 있다.
